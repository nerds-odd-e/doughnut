## Spacing Scale

Declared values (multiples of 4), aligned to notebook Settings panels:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tight inline gaps only if needed |
| sm | 8px | Action-bar control gaps (`gap-2`), leaf list row gaps |
| md | 16px | Nested collapse indent / content padding inside collapse |
| lg | 24px | Panel section padding (`p-6`), section/stack gaps (`mb-6`, `space-y-6`) |
| xl | 32px | Not required for Health panel |
| 2xl | 48px | Not required for Health panel |
| 3xl | 64px | Not required for Health panel |

**Exceptions:** Collapse titles use `py-3` (12px vertical) to match `AssimilationSettings` ÔÇö keep that live pattern even though 12 is not on the 8-point token table.

**Layout shell:** Health panel sits in the same `container mx-auto ÔÇŽ max-w-6xl` notebook page as Readme/Settings. No extra page margin system.

## Typography

Exactly three sizes and two weights, taken from notebook Settings / collapse patterns:

| Role | Size | Weight | Line Height | Maps to |
|------|------|--------|-------------|---------|
| Body | 14px | 400 | 1.5 | Idle prompt, empty-group message, leaf finding `label` (`text-sm`, default weight) |
| Label | 14px | 600 | 1.2 | Collapse group headers (`text-sm font-semibold` or `font-medium` Ôćĺ lock **600** for header emphasis) |
| Heading | 18px | 600 | 1.2 | Optional panel section title if used (`text-lg font-semibold`); tabs use existing `daisy-tab-lg` |

Do **not** add a fourth size or display type. Helper/muted copy (idle secondary line) stays **Body** at `text-base-content/70` (same as Settings descriptions).

**Severity:** Omit severity badges and severity text (D-13 + discretion: only `warning` exists; badges add noise).

## Interaction Contract

### Tab shell (HLTH-01)

| State | Behavior |
|-------|----------|
| Notebook page load | Active tab remains **readme**; Health not selected |
| Click Health | Show Health panel only; **no** lint API call; **no** mutation |
| Folder page | Health tab **absent** (`folder-workspace-tab-health` does not exist) |
| Panel switch | Explicit `v-if` / `v-else-if` for readme / settings / health ÔÇö never Settings via bare `v-else` |

### Run (HLTH-02, AFIX-01)

| State | Behavior |
|-------|----------|
| Idle (no successful report yet) | Idle prompt + action bar (Run lint + checkbox). No fabricated findings. `data-testid="notebook-health-idle"` |
| Click Run lint | `apiCallWithLoading(() => NotebookHealthController.lint({ path: { notebook } }))` ÔÇö bodyless, thin bar only (no `blockUi`) |
| Run in flight | Global thin loading bar; Run may stay enabled or use default button behavior ÔÇö do not invent a full-panel spinner |
| Run success | Replace idle with findings view; report ephemeral in component state |
| Re-Run | Replaces previous report (no history) |
| Run failure | Keep prior UI (idle or last report); error via existing `apiCallWithLoading` toast ÔÇö no inline error panel required |
| Checkbox on/off | Does **not** change request; never mutates notebook data |

### Findings (HLTH-03)

| Rule | Contract |
|------|----------|
| Groups | Render **all** `report.groups` (always-emit backend) |
| Header | `{{ group.title }} ({{ findingCount }})` where count = leaf items: `(items?.length ?? 0) + sum(children.items.length)` |
| Default open | Groups **with** findings expanded; groups with count `0` **collapsed** |
| Empty expanded | Quiet **No findings** (Body typography, muted) |
| Flat rules | `empty_folders`, `readme_only_folders`: list `item.label` only |
| Nested rule | `dead_wiki_links`: collapse per `child` (`child.title` = note title); leaves = token `item.label` |
| Labels | Text only (Vue default escaping; no `v-html`); **no** click-through |
| Client regroup | **Forbidden** ÔÇö wire shape only |

### Action bar

| Control | Placement | Phase 5 behavior |
|---------|-----------|------------------|
| Run lint | Primary, left or leading in flex row | Triggers lint only |
| Remove empty folders | Checkbox, same bar | UI-only local state; no Fix button |

Layout: `flex flex-wrap items-center gap-2` (or `gap-4`) at top of panel section ÔÇö visible in both idle and report views so Re-Run is always available.

## UI Considerations

> Shape-rooted UI state coverage for planner lift (`must_haves`). Empty/error **copy** is in Copywriting Contract; this section references those rows.

Applicable state considerations resolved: **8 covered, 2 backstop, 0 unresolved**

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| empty | Health findings list-collection | Ôťů covered | Before first successful Run, panel shows idle prompt (`notebook-health-idle`) and action bar ÔÇö no fabricated findings. After Run, groups with leaf count `0` stay collapsed; if expanded, show **No findings**. |
| empty | Remove empty folders checkbox (form control) | Ôťů covered | Checkbox defaults unchecked; empty means ÔÇťoption off,ÔÇŁ not a separate empty UI. |
| loading | Run lint interactive-control | Ôťů covered | In-flight uses existing thin-bar `apiCallWithLoading` only ÔÇö no panel skeleton, no `blockUi` modal. |
| error | Run lint / findings load | Ôťů covered | Failures surface via global toast from `apiCallWithLoading`; idle or last successful report remains; user retries with **Run lint**. |
| populated | Findings list-collection | Ôťů covered | All returned groups render; headers show title + leaf count; groups with findings expand by default; flat labels or nested noteÔćĺtoken collapses per wire shape. |
| partial | Report groups (mixed empty/non-empty rules) | Ôťů covered | Always render all groups; non-empty expanded, empty collapsed ÔÇö partial report is normal (e.g. only dead links). |
| overflow | Findings list-collection | ­čž¬ backstop | Long lists rely on normal page scroll (no inner max-height scroll region required). Verify visually / held-out that many findings remain reachable without clipping the action bar. |
| overflow | Tab nav | Ôťů covered | Existing `daisy-tabs-box` wraps with page width; third tab uses same tab component ÔÇö no new overflow treatment. |
| zero-one-many | Findings list-collection | Ôťů covered | Zero: idle or ÔÇťNo findingsÔÇŁ / count `(0)`. One: single leaf or single child note. Many: stacked collapses + leaf lists with `gap-2`; header count reflects leaf total. |
| long-text | Finding labels / note titles (static-content) | ­čž¬ backstop | Labels and note titles wrap as normal block text (no truncate/ellipsis required). Verify unusually long folder names / wiki tokens do not overflow the panel horizontally. |
