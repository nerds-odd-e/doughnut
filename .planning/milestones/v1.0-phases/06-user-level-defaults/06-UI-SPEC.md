# Phase 6 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Sources locked:** `06-CONTEXT.md` (D-01–D-15), `06-RESEARCH.md`, `05-UI-SPEC.md` / `05-CONTEXT.md` (Health chrome inherited), live `NotebookHealthPanel.vue`. Mode `--auto`: no user questions; CONTEXT discretion defaults applied.

**Relationship to Phase 5:** This contract **extends** Phase 5 Health chrome. Do **not** reinvent tabs, panel shell, findings, idle copy, Run lint styling, or spacing/typography/color tokens. Only the action-bar **prefill** and **Save as defaults** control are new this phase.

## Typography

Exactly three sizes and two weights — **identical to Phase 5**:

| Role | Size | Weight | Line Height | Maps to |
|------|------|--------|-------------|---------|
| Body | 14px | 400 | 1.5 | Idle prompt, empty-group message, leaf finding `label` (`text-sm`) |
| Label | 14px | 600 | 1.2 | Collapse group headers (`text-sm font-semibold`) |
| Heading | 18px | 600 | 1.2 | Optional panel section title if used (`text-lg font-semibold`); tabs use existing `daisy-tab-lg` |

Do **not** add a fourth size or display type. Save control label uses DaisyUI `daisy-btn-sm` default label typography (same family as Run lint), not a new type role.

**Severity:** Omit severity badges and severity text (inherited Phase 5).

## Color

DaisyUI semantic tokens only (no new hex palette). Ratio relative to Health panel surfaces — **identical to Phase 5**, with Save explicitly **not** accent:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `base-100` | Health panel section background (`bg-base-100`) |
| Secondary (30%) | `base-200` / `base-200/50` | Tab strip, collapse surfaces, borders `border-base-300` |
| Accent (10%) | `primary` | **Run lint** primary button only (`daisy-btn daisy-btn-primary daisy-btn-sm`) |
| Destructive | not used in Phase 6 | No Fix / purge / delete controls |

**Accent reserved for:** `Run lint` primary button only (unchanged from Phase 5).

**Not accent:** Save as defaults, tab active state, checkbox, collapse chrome, leaf labels, borders.

**Save control color:** Ghost secondary action — `daisy-btn daisy-btn-ghost daisy-btn-sm`. Must **not** use `daisy-btn-primary`, `daisy-btn-error`, `daisy-btn-warning`, or any destructive-looking treatment. Save must not read as Fix/purge.

**Text:** `text-base-content` for headers; `text-base-content/70` for idle/secondary body (unchanged).

## Interaction Contract

### Inherited (Phase 5 — do not regress)

| Area | Contract |
|------|----------|
| Tab shell | Health notebook-only; open Health alone does **not** call lint or mutate notebook data |
| Run lint | Bodyless `POST .../health/lint` via `apiCallWithLoading`; report-only; checkbox never changes lint request |
| Findings | Expandable groups from wire shape; labels text-only; no client regroup |
| Action bar visibility | Present in both idle and report views so Re-Run is always available |

### Prefill (DFLT-02, D-04–D-06)

| State | Behavior |
|-------|----------|
| Health panel shown | Seed **Remove empty folders** from injected `currentUser.healthRemoveEmptyFoldersDefault`. Prefer `onMounted` (panel remounts via `v-else-if="activeTab === 'health'"`). Fallback to `UserController.getUserProfile` only if preference missing on inject. |
| Unset / missing preference | Checkbox unchecked (`false`) — matches server default and Phase 5 initial local state |
| After prefill | Checkbox is **run-scoped local state**. User may toggle for this visit without overwriting saved defaults until explicit Save |
| Prefill side effects | **UI-only.** Opening Health, loading defaults, or prefill must **not** call lint, must **not** run Fix/purge, must **not** mutate notebook data |

### Save as defaults (DFLT-01, D-07–D-11)

| State | Behavior |
|-------|----------|
| Click **Save as defaults** | `apiCallWithLoading(() => UserController.updateUser({ path: { user }, body: UserDTO-shaped payload }))` — include `name`, `dailyAssimilationCount`, `spaceIntervals`, and `healthRemoveEmptyFoldersDefault: removeEmptyFolders` from current profile + checkbox. Thin bar only (no `blockUi`) |
| Save in flight | Global thin loading bar; do not invent a full-panel spinner or disable the whole Health panel |
| Save success | Update injected `currentUser` with returned `User` so other notebooks in the same session prefill the new value without full reload. Do **not** call lint; do **not** mutate notebook content |
| Save failure | Keep prior UI (checkbox local state unchanged relative to last user toggle); error via existing `apiCallWithLoading` toast — no inline error panel required. User can click **Save as defaults** again |
| Checkbox toggle alone | Does **not** PATCH; does **not** mutate notebook data |
| Cross-notebook | Save on notebook A, open Health on notebook B → checkbox reflects saved default (no notebook-scoped override UI) |

### Action bar layout (Phase 6)

| Control | Placement | Classes | Phase 6 behavior |
|---------|-----------|---------|------------------|
| Run lint | Leading (primary) | `daisy-btn daisy-btn-primary daisy-btn-sm` | Triggers lint only |
| Remove empty folders | Same bar, after Run | existing `CheckInput` wrapper | Prefill from user default; run-scoped toggle; no auto-save |
| Save as defaults | Same bar, after checkbox | `daisy-btn daisy-btn-ghost daisy-btn-sm` | Persists current checkbox value as user default only |

Layout: `flex flex-wrap items-center gap-2` (or `gap-4`) at top of panel — order **Run lint → Remove empty folders → Save as defaults**. Visible in both idle and report views.

**Visual hierarchy rule:** Primary accent stays exclusively on **Run lint**. Save is quieter (ghost) and must never be styled like a destructive Fix/purge action.

## UI Considerations

> Shape-rooted UI state coverage for planner lift (`must_haves`). Empty/error **copy** is in Copywriting Contract; this section references those rows.

Applicable state considerations resolved: **7 covered, 1 backstop, 0 unresolved** (Phase 6 delta elements; Phase 5 findings considerations remain in force via inheritance)

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| empty | Remove empty folders checkbox (form control) | ✅ covered | Unset user default and missing inject field both render unchecked (`false`). No separate empty-state message. |
| empty | Save as defaults (interactive-control) | ✅ covered | Always visible on action bar when Health is shown; no “no defaults yet” empty UI — empty preference is the unchecked checkbox. |
| loading | Save as defaults interactive-control | ✅ covered | In-flight uses existing thin-bar `apiCallWithLoading` only — no panel skeleton, no `blockUi` modal. |
| loading | Prefill on Health show | ✅ covered | Prefill is synchronous from injected `currentUser` (already loaded at app start). No loading UI for prefill. Profile fallback, if ever needed, uses the same thin-bar pattern only if an explicit fetch is required — default path has no spinner. |
| error | Save as defaults | ✅ covered | Failures surface via global toast from `apiCallWithLoading`; checkbox and panel remain; user retries **Save as defaults**. |
| populated | Prefill from saved default | ✅ covered | When `healthRemoveEmptyFoldersDefault` is `true`, checkbox is checked on Health show; when `false`, unchecked. |
| partial | Save PATCH body fields | ✅ covered | Not a partial form UI — Save always sends full UserDTO-shaped body (`name`, existing prefs, new boolean) so omitted fields do not reset other preferences. |
| overflow | Action bar (nav / controls) | 🧪 backstop | `flex-wrap` on action bar keeps Run, checkbox, and Save reachable on narrow widths. Verify visually that Save remains tappable without horizontal clip. |
| zero-one-many | Findings list-collection | ✅ covered | Inherited Phase 5 — unchanged this phase. |
| long-text | Save as defaults label | ✅ covered | Fixed short label **Save as defaults**; no dynamic long text. Button uses `daisy-btn-sm` and wraps with action bar `flex-wrap` if needed. |

## Out of Scope (UI)

Do not design or implement in Phase 6:

- Fix / Apply button or purge confirmation (Phase 7)
- Health defaults fields in global User Settings / `UserProfileForm`
- Per-notebook overrides of user defaults (v2 DFLT-10)
- Auto-save on every checkbox toggle
- Severity badges or severity settings UI
- Click-through to editor or folder (v2 HLTH-11)
- Health summary chip on notebook chrome (v2 HLTH-10)
- Health on folder settings
- Dedicated `/health` route or findings dialog
- New design tokens, themes, or icon sets
- Sending defaults on the lint request body
