# Phase 6 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Sources locked:** `06-CONTEXT.md` (D-05, D-06, D-07), `06-RESEARCH.md` (retry remount + alert recommendations), `ROADMAP.md` Phase 6 / OVL-01, live `AnsweredSpellingQuestion.vue` / `RecallPage.vue` / DaisyUI recall alerts. Mode `--auto`: no interactive prompts; discretionary choices logged below.

**Product locks (do not contradict):**
- **D-05:** Do **not** advance the recall queue; stay on the same memory tracker and allow retry
- **D-06:** Distinct try-again alert — warning-style chrome; not success / plain incorrect / accidental-match copy
- **D-07:** No matched-notes section / no offer-link CTAs for OVERLAP; `matchedNotes` stays empty; wire on `outcome === 'OVERLAP'` / `overlap === true`

## Color

DaisyUI semantic tokens only (no new hex palette). Ratio relative to the spelling answer-result surface:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `base-100` | Result surface / page background behind the stack |
| Secondary (30%) | `base-200` / `border-base-300` | Existing `NoteUnderQuestion` chrome; secondary CTA chrome for Try again |
| Accent (10%) | `primary` | Reserved for existing primary recall actions (e.g. spelling Answer submit, ViewMemoryTrackerLink) — **not** the OVERLAP Try again button |
| Warning (outcome) | `warning` (`daisy-alert-warning`) | OVERLAP try-again alert only (D-06) |
| Destructive / miss | `error` (`daisy-alert-error`) | Plain incorrect + accidental-match alerts (unchanged) |
| Success | `success` (`daisy-alert-success`) | Correct answers only when `outcome !== 'OVERLAP'` (unchanged) |

**Accent reserved for:** Existing primary spelling Answer submit and other pre-existing primary CTAs — **not** the Phase 6 Try again control.

**Try again button color:** `daisy-btn daisy-btn-secondary daisy-btn-sm` — secondary family (same as Phase 4 “Link to this note”), so it reads as a next-step control without competing with primary Answer / memory-tracker actions.

**Alert styling (D-06):**
- Correct (`correct === true`, outcome ≠ `OVERLAP`) → `daisy-alert-success` (unchanged)
- Plain wrong (`correct === false`, outcome ≠ `ACCIDENTAL_MATCH` and ≠ `OVERLAP`) → `daisy-alert-error` (unchanged)
- Accidental match (`outcome === 'ACCIDENTAL_MATCH'`) → `daisy-alert-error` + existing AM copy (unchanged)
- Overlap (`outcome === 'OVERLAP'` or `overlap === true`) → `daisy-alert-warning` + try-again copy below — **never** success green, **never** error red

**Text:** Alert text inherits DaisyUI alert content color; CTA uses DaisyUI button content color.

**Primary screen:** Spelling answer result when `outcome === 'OVERLAP'` (`AnsweredSpellingQuestion`).

## Interaction Contract

### Alert branch (D-06)

| Condition | Alert class | Copy source | testid |
|-----------|-------------|-------------|--------|
| `answer.correct === true` and outcome ≠ `OVERLAP` | `daisy-alert-success` | `Correct!` (unchanged) | — |
| `correct === false` and outcome ≠ `ACCIDENTAL_MATCH` and ≠ `OVERLAP` | `daisy-alert-error` | `Your answer \`${spellingAnswer}\` is incorrect.` (unchanged) | — |
| `outcome === 'ACCIDENTAL_MATCH'` | `daisy-alert-error` | Existing AM string (unchanged) | `accidental-match-alert` |
| `outcome === 'OVERLAP'` (or `overlap === true`) | `daisy-alert-warning` | Overlap try-again string below | `overlap-try-again-alert` |

**Correct-path guard:** Even if backend sends `correct: false` for OVERLAP (D-03), frontend alert branch must key off `outcome === 'OVERLAP'` / `overlap === true` **before** treating the result as a plain miss — so OVERLAP never paints `daisy-alert-error` or success green.

### Queue / retry (D-05)

| Step | Behavior |
|------|----------|
| Receive OVERLAP from `@answered` | Push into `previousAnsweredQuestions`; `viewLastAnsweredQuestion` to show result; **skip** `moveToNextMemoryTracker`; **skip** `getThresholdExceeded` / re-assimilation |
| Show result | `AnsweredSpellingQuestion` with warning alert + Try again; reviewed note stack unchanged; no matched-notes |
| User clicks **Try again** | Clear `previousAnsweredQuestionCursor` (resume quiz); increment `spellingRetryNonce`; same `currentIndex` / memory tracker |
| Remount | Quiz spelling key includes nonce → `SpellingQuestionDisplay` remounts → `askAQuestion` / `getSpellingQuestion` allocates a **new** unanswered prompt |
| User answers distinguishing alias | Existing CORRECT path (advance + credit) — unchanged |
| User answers shared/non-distinguishing again | OVERLAP again — stay-and-retry repeats |

**Anti-pattern (forbidden):** Relying only on progress-bar resume without remount nonce — same `:key="spelling-${memoryTrackerId}"` leaves the prior answered prompt mounted and retries fail with “already answered.”

### Matched notes / link CTAs (D-07)

| State | Behavior |
|-------|----------|
| OVERLAP | Matched section **absent**; no link CTAs; ignore any accidental `matchedNotes` payload for UI (backend should leave empty) |
| ACCIDENTAL_MATCH | Existing Phase 3–4 behavior unchanged |
| Plain wrong / correct | Unchanged |

## must_haves (planner lift)

Truths the executor and verifier must treat as non-negotiable for OVL-01 UI:

1. When `outcome === 'OVERLAP'`, the user sees a **warning** alert (`daisy-alert-warning`) with copy **Correct, but we're looking for another answer — try again.** (`data-testid="overlap-try-again-alert"`).
2. OVERLAP does **not** advance the recall queue; the same memory tracker remains current.
3. OVERLAP does **not** open the matched-notes section or any offer-link CTA.
4. A visible **Try again** control (`data-testid="overlap-try-again"`) returns the user to a **fresh** spelling prompt for the same tracker (remount nonce required).
5. OVERLAP does **not** trigger the wrong-answer re-assimilation threshold UI.
6. Correct / plain incorrect / accidental-match alert chrome and copy remain unchanged for non-OVERLAP outcomes.
