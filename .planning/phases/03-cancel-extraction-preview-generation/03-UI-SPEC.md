# Phase 3 — UI Design Contract

> Visual and interaction contract for cancelable extraction-preview generation. Generated by gsd-ui-researcher; pending gsd-ui-checker verification.

---

## Phase Boundary

Phase 3 is the second Behavior adoption of the Phase 1 cancel contract: AI extraction-preview generation uses the global blocking spinner with Cancel. After cancel, layout selections stay intact; Extract-from-layout cancel never opens the preview; in-preview Ask-AI-to-retry cancel keeps the prior preview content.

This contract **adopts** Phase 1 Cancel visuals unchanged (same stance as Phase 2). It does **not** redesign `LoadingModal`, Overlay, Cancel label/placement/a11y, Escape/backdrop cancel, add a layout empty-state after preview cancel, or make create-note cancelable (Phase 4 / REFN-05).

Sources locked under `--auto`: Phase 1 `01-UI-SPEC.md`, Phase 2 `02-UI-SPEC.md` adoption stance, `03-CONTEXT.md` (D-01–D-10), `03-RESEARCH.md`, REQUIREMENTS REFN-03–04, ROADMAP Phase 3 success criteria.

## Surfaces in Scope

### 1. Global blocking spinner (Cancel) — ADOPT ONLY

Reuse Phase 1 `01-UI-SPEC.md` Blocking Surface Contract verbatim for the extraction-preview generation blocker:

- Full-viewport overlay, white spinner, white message, optional neutral `Cancel` beneath the message.
- Cancel: native `<button type="button">`, accessible name `Cancel`, `daisy-btn daisy-btn-ghost text-white`, 40px height, 16px inline padding, 14px/600 label, white focus outline; no icon; not destructive.
- No Escape/backdrop cancel, no autofocus, no toast/`Cancelling...`/`Cancelled` interstitial.
- Preview call must opt in with `{ blockUi: true, cancelable: true }` and message `AI is generating preview...` (D-08, D-09, D-10).

Do not change Cancel copy, spacing, color, or motion in this phase.

### 2. Post-cancel domain landings — EXISTING CONTROLS ONLY (differs from Phase 2)

Phase 3 introduces **no new empty-state or retry panel**. Selections are preserved (D-02, D-04), so recovery reuses controls already on the layout and preview surfaces.

| Path | Post-cancel surface | Required state | Retry affordance |
|------|---------------------|----------------|------------------|
| Extract-from-layout cancel | Populated `data-test-id="refinement-layout"` list | Same checkboxes / `selectedItemIds` as before Extract; `showExtractionPreview === false`; preview panel absent | Existing primary `Extract` (`data-test-id="extract-refinement-layout"`) — remains enabled because selection is non-empty |
| In-preview Ask-AI-to-retry cancel | Existing `data-test-id="extraction-preview"` panel | Prior preview field values unchanged; `createError` not set by cancel; Back / Ask AI to retry / Create note remain as today | Existing ghost `Ask AI to retry` (`data-test-id="retry-extraction-preview"`); unsaved-edit confirm before retry stays (D-07) |

Do **not**:

- Show `refinement-layout-empty` or invent a layout-panel empty state for preview cancel (unlike Phase 2 layout cancel).
- Clear selection, reload layout, emit `contentUpdated`, mutate note content, or apply cancelled/late preview payload (D-02, D-05).
- Wipe preview fields, force Back to layout, or open preview via the API-error force-show path on cancel (D-01, D-06).
- Auto-close the Refine-note dialog (`AssimilationSettings`).

### 3. Out of scope (explicit non-surfaces)

| Surface | Phase 3 treatment |
|---------|-------------------|
| Create-note blocker (`AI is creating note...`) | Remains noncancelable; Cancel absent (Phase 4 / REFN-05) |
| Layout generation blocker | Already cancelable from Phase 2; leave untouched |
| Remove-selected / export dialogs | Unchanged |

## Copywriting Contract

| Element | Copy | Source |
|---------|------|--------|
| Conditional blocker action | `Cancel` | Phase 1 locked — ADOPT ONLY; do not change |
| Preview loading message | `AI is generating preview...` | D-08 / existing product message |
| Loading message fallback | `Processing...` | Unchanged global default |
| Primary retry CTA (Extract-from-layout cancel) | `Extract` | Existing layout action; D-04 — no new empty-state CTA |
| Primary retry CTA (in-preview regenerate cancel) | `Ask AI to retry` | Existing preview action; D-06 |
| Empty state heading | Not applicable | No new empty panel in this phase (D-04) |
| Empty state body | Not applicable | Selections preserved; populated layout remains visible |
| Error state (accepted cancellation) | None — no toast, banner, `createError`, or interstitial | D-03 / Phase 1 D-11 |
| Error state (ordinary API failure) | Existing `createError` / force-show preview path — out of cancel scope; cancel must not share it | RESEARCH / discretion |
| Destructive confirmation | None for Cancel | Cancel is not destructive |
| Unsaved-edit confirm before retry | Existing: `You have unsaved edits to the extract preview. Ask AI to retry will discard your edits and regenerate the preview.` | D-07 — unchanged |

Do not substitute Cancel with `Stop`, `Stop generation`, or `Cancel request`. Do not invent layout empty-state copy or alternate retry labels (`Try again`, `Regenerate preview`) in this phase.

Test selectors (contractual):

- Overlay Cancel: accessible / visible name `Cancel` (Phase 1).
- Layout retry after Extract cancel: `data-test-id="extract-refinement-layout"` (enabled with preserved selection).
- In-preview regenerate: `data-test-id="retry-extraction-preview"`.
- Preview panel: `data-test-id="extraction-preview"` — must be **absent** after Extract-from-layout cancel; must **remain** after in-preview regenerate cancel.
- Layout panel: `data-test-id="refinement-layout"` — must remain visible after Extract-from-layout cancel.
- Do **not** expect `refinement-layout-empty` as the Extract-cancel recovery surface.

## Interaction Contract

| Trigger | Required outcome |
|---------|------------------|
| Preview pending (Extract or Ask AI to retry) with opt-in cancelable blocker | Global spinner + `AI is generating preview...` + `Cancel` |
| User activates Cancel from Extract-from-layout | Abort only that request; remove only that loading state; no toast; no `createError`; no `contentUpdated`; no navigation; dialog stays open; stay on layout; selections unchanged; preview never shown; Extract remains enabled |
| User activates Cancel during in-preview Ask AI to retry | Same silent abort/cleanup; remain on `extraction-preview` with prior field values; do not wipe fields or force Back |
| User activates `Extract` after Extract-path cancel | Starts a fresh cancelable preview request (same loading + Cancel contract) |
| User activates `Ask AI to retry` after regenerate cancel | Existing unsaved-edit confirm (if edits), then fresh cancelable preview request |
| Late/partial response after cancel | Must not apply preview payload or open/change preview (D-05) |
| Create-note pending | Shared blocker without Cancel (out of scope / Phase 4) |
| Concurrent other loading states | Survive preview cancel; Cancel identity-bound per Phase 1 |

### Visual hierarchy / focal points

| Moment | First focal point | Secondary |
|--------|-------------------|-----------|
| Preview pending | Global spinner + message `AI is generating preview...`, then Cancel beneath | Refine dialog content obscured by overlay |
| After Extract-from-layout cancel | Populated Note layout list with preserved checks | Primary `Extract` as retry CTA |
| After in-preview regenerate cancel | Extract preview fields with prior content | Ghost `Ask AI to retry` as regenerate CTA |

| Category | Element(s) | Status | Resolution / Reason |

| loading | Global blocking surface (preview generation) | ✅ covered | Pending preview shows existing overlay + message `AI is generating preview...` + Phase 1 Cancel when opted in. |
| empty | Layout / preview post-cancel | ✅ covered | No new empty panel. Extract-cancel keeps populated layout + selection; regenerate-cancel keeps prior preview content. Phase 2 empty panel is not reused for this path (D-04). |

### Planner lift hints (`must_haves`)

- While extraction-preview generation is pending, the global blocker shows `AI is generating preview...` and an accessible `Cancel` control.
- Activating Cancel from Extract removes that blocker promptly, shows no error toast, does not open `extraction-preview`, keeps layout selections, and leaves the Refine-note dialog open with Extract still enabled.
- Activating Cancel during Ask AI to retry keeps `extraction-preview` visible with prior field values and shows no error UI.
- Create-note blocker remains visually/behaviorally noncancelable in this phase (no Cancel on that path).
