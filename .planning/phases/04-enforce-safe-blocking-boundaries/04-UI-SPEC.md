# Phase 4 — UI Design Contract

> Visual and interaction contract for enforcing safe blocking boundaries (create-note Cancel-absent + intentionally noncancelable chrome). Generated by gsd-ui-researcher; pending gsd-ui-checker verification.

---

## Phase Boundary

Phase 4 is a Behavior close-out, not a Cancel redesign. The observable UI change is **negative space**: while final extracted-note creation is pending, the shared global blocker shows `AI is creating note...` and **must not** expose Cancel (REFN-05 / D-01). Layout and extraction-preview Cancel behavior stays exactly as Phases 2–3 (D-06).

This contract **adopts** Phase 1 Cancel visuals unchanged for the two cancelable adopters, and documents that all other whole-UI blockers keep the existing spinner-without-Cancel chrome (no new controls). It does **not** redesign `LoadingModal`, Overlay, Cancel label/placement/a11y, Escape/backdrop cancel, add Cancel to create-note or other mutations, invent retry-after-abort for create, or claim server-side stop.

Sources locked under `--auto`: Phase 1 `01-UI-SPEC.md` (Cancel visual contract — ADOPT), Phase 3 `03-UI-SPEC.md` (adoption stance), `04-CONTEXT.md` (D-01–D-11), `04-RESEARCH.md`, REQUIREMENTS REFN-05 / COHE-02, `.cursor/rules/frontend-api.mdc`, `LoadingModal.vue`.

## Surfaces in Scope

### 1. Create-note noncancelable blocker — ENFORCE (REFN-05)

While `createExtractedNote` is pending, the global blocker must match Phase 1 **noncancelable selected-blocker** rendering:

- Full-viewport overlay (`z-index: 10000`), white spinner, white message `AI is creating note...`.
- **Cancel is absent from the DOM** and reserves no space (no placeholder, empty action row, or added gap).
- No Escape/backdrop cancel, no autofocus target for cancel, no toast/`Cancelling...`/`Cancelled` interstitial, no retry-after-abort affordance for create (D-02).
- Caller must **not** opt into `{ cancelable: true }`; keep noncancelable `runWithBlockingApiLoading` (or equivalent noncancelable `{ blockUi: true }` only) — D-04.
- Success path unchanged: one successful completion still focuses/navigates via `focusNoteRealm` as today (D-02). Ordinary create failures keep existing `createError` / stay-on-preview behavior — out of cancel scope.

This is intentional safety, not a missing opt-in (D-01).

### 2. Cancelable layout + preview blockers — ADOPT ONLY (unchanged)

Reuse Phase 1 `01-UI-SPEC.md` Blocking Surface Contract verbatim for the two in-scope cancelable adopters (D-06):

| Operation | Message | Cancel |
|-----------|---------|--------|
| Layout generation | `AI is generating layout...` | Present — Phase 1 Cancel |
| Extraction-preview generation | `AI is generating preview...` | Present — Phase 1 Cancel |

- Cancel: native `<button type="button">`, accessible name `Cancel`, `daisy-btn daisy-btn-ghost text-white`, 40px height, 16px inline padding, 14px/600 label, white focus outline; no icon; not destructive.
- Post-cancel domain landings remain Phase 2 / Phase 3 contracts — do not reopen.

Do not change Cancel copy, spacing, color, or motion in this phase.

### 3. Intentionally noncancelable whole-UI blockers — SAME CHROME (COHE-02 / D-07)

All other whole-UI blockers keep the **existing spinner-without-Cancel** chrome. No new controls, alternate messages for “safety,” or Cancel migration in this phase. Broader adoption is `ADPT-01` / v2.

Representative messages (existing product copy — do not invent new wording here):

| Operation | Message | Cancel |
|-----------|---------|--------|
| Create extracted note | `AI is creating note...` | Absent |
| Remove refinement content | `AI is removing content...` | Absent |
| Assimilate unit | `Assimilating...` | Absent |
| Load next assimilation note | `Loading next note...` | Absent |
| Book layout mutations | `Updating book layout…` | Absent |
| Book layout AI suggest | `Analyzing book layout…` | Absent (ADPT-01 candidate) |
| Book layout apply suggestion | `Applying layout changes…` | Absent |
| Note delete / reduce | `Deleting note...` / `Reducing to source property...` | Absent |
| Relationship finalize | `Creating relationship note...` | Absent |
| Attach book upload | `Uploading book…` | Absent |

Visual rule: when the selected global state is noncancelable, `LoadingModal` renders overlay + spinner + message only (`v-if="cancelControl"` false) — identical to Phase 1 noncancelable row.

### 4. Nonblocking / out of Cancel UX (D-08)

Thin-bar or direct-SDK note-refinement helpers (e.g. export request fetches without `blockUi`) are **nonblocking**: no whole-UI modal, no Cancel chrome. Do not invent a Cancel surface for them.

### 5. Out of scope (explicit non-surfaces)

| Surface | Phase 4 treatment |
|---------|-------------------|
| `LoadingModal` / Overlay / Cancel button redesign | Unchanged — ADOPT Phase 1 |
| Escape / backdrop cancel | Not added |
| Create-note Cancel | Forbidden (D-04) |
| Cancel on other mutations / book-layout AI | Deferred ADPT-01 |
| Server-cooperative / mutation-safe cancel UI | Deferred SERV-01/02 |
| New empty / retry panels for create | None — create is not cancelable |

## Copywriting Contract

| Element | Copy | Source |
|---------|------|--------|
| Create-note loading message | `AI is creating note...` | D-01 / existing product — LOCKED |
| Conditional blocker action (layout/preview) | `Cancel` | Phase 1 locked — ADOPT ONLY |
| Layout loading message | `AI is generating layout...` | Phase 2 — unchanged |
| Preview loading message | `AI is generating preview...` | Phase 3 — unchanged |
| Loading message fallback | `Processing...` | Unchanged global default |
| Primary CTA (create pending) | None on the blocker; product CTA remains existing `Create note` before/after the wait | D-02 — no Cancel / no retry-after-abort CTA |
| Empty state heading | Not applicable | No empty surface introduced |
| Empty state body | Not applicable | Create is not cancelable; no post-cancel empty state |
| Error state (accepted cancellation) | N/A on create path — Cancel must not appear | D-01 |
| Error state (ordinary create API failure) | Existing `createError` banner on preview — unchanged; not a cancel path | D-02 / RESEARCH |
| Destructive confirmation | None for this phase's blocker boundary | Create Cancel is forbidden, not confirmed |

Do not substitute create's missing Cancel with `Stop`, `Please wait`, or explanatory safety copy under the spinner. Absence of Cancel **is** the safety signal.

Test selectors / assertions (contractual):

- Overlay Cancel: accessible / visible name `Cancel` — must be **absent** while message is `AI is creating note...`.
- Layout / preview pending: Cancel **present** with messages `AI is generating layout...` / `AI is generating preview...` (Phase 2/3 regression).
- Create success: existing focus/navigation via `focusNoteRealm` — no new UI chrome.

## Interaction Contract

| Trigger | Required outcome |
|---------|------------------|
| User activates `Create note` with pending create | Global spinner + `AI is creating note...`; **no Cancel** in DOM; no Escape/backdrop cancel |
| Create succeeds | Existing focus/navigate via `focusNoteRealm`; overlay clears; no cancellation interstitial |
| Create fails (ordinary API error) | Existing `createError` / stay-on-preview path; overlay clears; still no Cancel was shown |
| Layout / preview pending | Unchanged Phase 2/3: spinner + message + Cancel |
| User cancels layout / preview | Unchanged Phase 2/3 post-cancel landings |
| Any intentionally noncancelable whole-UI blocker (D-07 inventory) | Spinner + its existing message; Cancel absent; no new controls |
| Nonblocking export / thin-bar helpers | No whole-UI modal; no Cancel chrome |
| Concurrent blockers | Phase 1 identity-bound selection: Cancel only if **selected** state is cancelable; create selected ⇒ Cancel absent even if an older cancelable state exists under the stack |

### Visual hierarchy / focal points

| Moment | First focal point | Secondary |
|--------|-------------------|-----------|
| Create-note pending | Global spinner + message `AI is creating note...` only | Refine dialog content obscured; **no** action control under the message |
| Layout / preview pending | Spinner + message, then Cancel beneath | Unchanged from Phases 2–3 |
| Intentionally noncancelable blocker elsewhere | Spinner + existing operation message only | Same negative-space rule as create |
