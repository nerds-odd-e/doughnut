# Phase 2 — UI Design Contract

> Visual and interaction contract for cancelable refinement-layout generation. Generated by gsd-ui-researcher; pending gsd-ui-checker verification.

---

## Phase Boundary

Phase 2 is the first Behavior adoption of the Phase 1 cancel contract: initial AI refinement-layout generation uses the global blocking spinner with Cancel, and after cancel (or empty layout) the Refine-note dialog stays open with an in-dialog retry control.

This contract **adopts** Phase 1 Cancel visuals unchanged. It does **not** redesign `LoadingModal`, Overlay, Cancel label/placement/a11y, Escape/backdrop cancel, extract-preview cancellation (Phase 3), or create-note cancelability (Phase 4).

Sources locked under `--auto`: Phase 1 `01-UI-SPEC.md`, `01-CONTEXT.md` (D-11–D-12), `02-RESEARCH.md`, REQUIREMENTS CANC-01–04 / REFN-01–02, ROADMAP Phase 2 success criteria.

## Surfaces in Scope

### 1. Global blocking spinner (Cancel) — ADOPT ONLY

Reuse Phase 1 `01-UI-SPEC.md` Blocking Surface Contract verbatim for the layout-generation blocker:

- Full-viewport overlay, white spinner, white message, optional neutral `Cancel` beneath the message.
- Cancel: native `<button type="button">`, accessible name `Cancel`, `daisy-btn daisy-btn-ghost text-white`, 40px height, 16px inline padding, 14px/600 label, white focus outline; no icon; not destructive.
- No Escape/backdrop cancel, no autofocus, no toast/`Cancelling...`/`Cancelled` interstitial.
- Layout call must opt in with `{ blockUi: true, cancelable: true }` and message `AI is generating layout...`.

Do not change Cancel copy, spacing, color, or motion in this phase.

### 2. Empty / cancelled layout retry panel — NEW (domain post-cancel)

When `refinementLayoutItems.length === 0` after a layout attempt (user cancel, empty completed result, or failed load that cleared items), render a compact panel inside `NoteRefinement` so the user is not left with only dialog `Close`.

| Property | Contract |
|----------|----------|
| Placement | Inside the Refine-note modal (`AssimilationSettings`), as the NoteRefinement body above the modal `Close` action |
| Visibility | Shown whenever there are zero layout items **and** the global layout blocker is not the selected surface (retry is dialog-local, not on the overlay) |
| Container | Match populated layout shell: `rounded-lg bg-accent p-4` (and `mb-4` as today), `data-test-id="refinement-layout-empty"` |
| Heading | `Note layout:` — same section label as the populated panel (`font-semibold`, `text-accent-content`, `mb-3`) |
| Body copy | None required — next step is the retry CTA (mirrors extract-preview: section label + actions, no invented empty-paragraph brand language) |
| Primary action | Ghost small DaisyUI button: visible text `Ask AI to retry`, `data-test-id="retry-refinement-layout"`, class equivalent of `daisy-btn daisy-btn-ghost daisy-btn-sm` |
| Action behavior | Invokes the same `loadRefinementLayout` path (fresh cancelable blocking request). Does not close the dialog, emit `contentUpdated`, or mutate note content |
| Populated transition | When items arrive, replace empty panel with existing `data-test-id="refinement-layout"` list UI; do not keep both |
| Dialog shell | Parent `Close` remains available; cancel must not auto-close the refine modal |

Shared empty panel for cancel, empty AI success (zero items), and cleared-after-error — one surface, one retry control (RESEARCH discretion A2).

## Copywriting Contract

| Element | Copy | Source |
|---------|------|--------|
| Conditional blocker action | `Cancel` | Phase 1 locked — do not change |
| Layout loading message | `AI is generating layout...` | RESEARCH recommendation; sibling `AI is …` messages |
| Loading message fallback | `Processing...` | Unchanged global default |
| Primary CTA (empty/cancelled layout) | `Ask AI to retry` | Existing extract-preview product pattern |
| Empty state heading | `Note layout:` | Existing populated section label |
| Empty state body | _(none)_ | Extract-preview pattern: heading + actions without invented explanatory paragraph |

| Destructive confirmation | None for Cancel or layout retry | Cancel is not destructive |

- Empty retry: `data-test-id="retry-refinement-layout"` (distinct from `retry-extraction-preview`).
- Empty panel root: `data-test-id="refinement-layout-empty"`.

## Interaction Contract

| Trigger | Required outcome |
|---------|------------------|
| Layout pending with opt-in cancelable blocker | Global spinner + `AI is generating layout...` + `Cancel` |
| User activates Cancel | Abort only that request; remove only that loading state; no toast; no `contentUpdated`; no navigation; dialog stays open; empty retry panel visible |
| User activates `Ask AI to retry` | Starts a fresh cancelable layout request (same loading + Cancel contract) |
| Layout completes with ≥1 items | Populated `refinement-layout` panel; empty panel absent |
| Layout completes with 0 items | Same empty retry panel as cancel |
| Concurrent other loading states | Survive layout cancel (CANC-04); Cancel identity-bound per Phase 1 |

### Planner lift hints (`must_haves`)

- While initial layout generation is pending, the global blocker shows `AI is generating layout...` and an accessible `Cancel` control.
- Activating Cancel removes that blocker promptly, shows no error toast, does not emit note-content updates, and leaves the Refine-note dialog open.
- After cancel (and after empty layout), `data-test-id="retry-refinement-layout"` is present and starts a fresh layout request.
- Extract-preview and create-note blockers remain visually/behaviorally unchanged in this phase (no Cancel on those paths yet).
