<objective>
Extend the backend→frontend answer **contract** (OpenAPI types/schema) so it can represent a third outcome — an accidental match carrying a matched-note id, plus an overlap flag and matched-note topology — and regenerate the TypeScript client so the existing frontend type-checks. This is a pure **Structure** phase: the new states must be **representable** in the contract but **not yet returned** by any backend code path. No new service, no new endpoint, no new grading logic, no Flyway migration, no UI.

Purpose: Lay the contract foundation that Phases 2 (accidental-match grading), 3 (reveal both notes), 4 (offer link), 5 (overlap declaration), and 6 (overlap "try again") will wire behavior against. Proving the contract round-trip end-to-end now (entity → enum → DTO → regen → frontend type-check → backend no-behavior test) catches the highest-risk assumption — that a JPA `@Transient` field surfaces in the springdoc OpenAPI schema — after one commit instead of after later phases have built on a silent gap.

Output: One new enum (`AnswerOutcome`), additive `@Transient` fields on `Answer`, additive optional fields on `AnsweredQuestion`, regenerated `types.gen.ts`/`sdk.gen.ts`/`open_api_docs.yaml`, and two extended backend controller tests asserting the new fields stay null on the existing paths.
</objective>
