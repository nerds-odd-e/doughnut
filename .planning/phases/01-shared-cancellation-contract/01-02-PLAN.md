<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Rendered Cancel control → selected loading state | The event must remain bound to the state represented when the button subtree was rendered. |
| Browser cancellation UI → user interpretation | The control abandons the browser wait and must not imply cooperative server termination. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-01-01 | Denial of Service | `LoadingModal` action binding | high | mitigate | Key by selected id, pass its closure directly, and test repeated captured action against a revealed replacement. |
| T-01-02 | Tampering / Denial of Service | `currentBlockingApiState` projection | high | mitigate | Derive message/id/action from one selected object and test that other active states survive. |
| T-01-03 | Tampering | Late operation success after accepted cancel | high | mitigate | Consume Plan 01's accepted-latch result only; UI never owns or recreates terminal classification. |
| T-01-04 | Repudiation / integrity ambiguity | Cancel label and feedback | medium | mitigate | Fixed neutral `Cancel` copy, no `Stop` wording or server claim, no success/error/interstitial, and no unsafe product adoption. |
</threat_model>
