# Phase 1 — UI Design Contract

> Visual and interaction contract for the shared cancellation structure. Generated by gsd-ui-researcher; pending gsd-ui-checker verification.

---

## Phase Boundary

Phase 1 introduces no user-visible behavior change. Every existing blocker must retain its current overlay, spinner, message, stacking, and dismissal behavior and must render no Cancel control. The phase may add the dormant conditional-control path required by Phase 2, but no product call site opts into it yet.

This contract does not redesign the blocker, add progress reporting, claim that server work stopped, or add cancellation to transactional mutations.

## Design System

| Property | Value |
|----------|-------|
| Tool | Tailwind CSS 4.3.3 + DaisyUI 5.6.18 |
| Preset | DaisyUI `light` default and `dark` preferred-color-scheme themes |
| Component library | DaisyUI plus existing Doughnut Vue components (`Overlay.vue`, `LoadingModal.vue`) |
| Icon library | Lucide Vue; no icon is used for Cancel |
| Font | Inherited Tailwind system sans stack: `ui-sans-serif, system-ui, sans-serif` |

`components.json` is absent. shadcn is not applicable to this Vue/DaisyUI project, and this phase adds no dependency, registry block, or custom visual system.

---

## Blocking Surface Contract

Preserve the existing global surface from `LoadingModal.vue` and `Overlay.vue`:

- Full-viewport fixed overlay at `z-index: 10000`, centered in both axes, with `rgba(0, 0, 0, 0.70)` background.
- White `daisy-loading daisy-loading-spinner daisy-loading-lg` spinner.
- Caller-supplied loading message, falling back to `Processing...`, in white at 18px, weight 500, with zero margin.
- Spinner and message remain a centered vertical stack with 16px gaps. The conditional control joins that same stack; do not add a card, title, close icon, progress bar, percentage, or explanatory copy.
- Preserve the current overlay and spinner motion. Do not add a separate transition for Cancel, a `Cancelling...` state, or a close-and-reopen animation when blockers change.

### Selected-blocker states

| Selected global state | Required rendering | Required transition |
|-----------------------|--------------------|---------------------|
| No blocking state | No global blocker | Unchanged current behavior |
| Newest blocker is noncancelable | Existing overlay + spinner + its message; Cancel is absent from the DOM and reserves no space | Unchanged current behavior |
| Newest blocker is cancelable | Same overlay + spinner + message, with one neutral `Cancel` button beneath the message | Control targets only this state's identity |
| Cancelled blocker has no older blocker | Remove the overlay immediately | No success, error, or cancellation interstitial |
| Cancelled blocker reveals an older blocker | Keep the overlay mounted and immediately replace message and conditional control from the older state | No close-and-reopen transition; spinner remains visible |
| Newest blocker is noncancelable and an older hidden blocker is cancelable | Render no Cancel control | Hidden capability must not leak into the selected surface |

### Conditional Cancel control

- Render only when the selected loading state exposes its own cancel action. Conditional absence means no placeholder, empty action row, or added gap.
- Use a native `<button type="button">` with visible text and accessible name `Cancel`.
- Visual treatment: neutral ghost button, no icon, no destructive/error color. Use the equivalent of `daisy-btn daisy-btn-ghost text-white`; keep the DaisyUI 40px height, 16px inline padding, 14px/600 label, and 2px visible focus outline. Hover/focus may use at most a 10% white translucent fill; focus outline remains white against the dark overlay.
- Click, Enter, and Space activate the state-bound action once. Repeated activation or a stale callback must not target a replacement blocker.
- Do not autofocus Cancel, bind cancellation to Escape, or make backdrop clicks cancel. Phase 1 adds no broader focus-management or dialog-semantics redesign.
- Acceptance is acknowledged only by the immediate blocker disappearance or revelation described above. Do not render a toast, banner, `Cancelling...`, `Cancelled`, disabled-button wait state, confirmation dialog, console message, or telemetry event.
- If the selected state changes while the overlay stays mounted, the button subtree is identity-bound to that state so its accessible action and callback update together.

## Spacing Scale

Declared values (all layout spacing remains on the existing 4px/8px scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Small internal alignment only; not newly used on the blocker |
| sm | 8px | Compact control spacing; not newly used on the blocker |
| md | 16px | Existing spinner/message/action stack gap and Cancel inline padding |
| lg | 24px | Inherited section spacing; not newly used on the blocker |
| xl | 32px | Inherited layout gap; not newly used on the blocker |
| 2xl | 48px | Inherited major section break; not newly used on the blocker |
| 3xl | 64px | Inherited page-level spacing; not newly used on the blocker |

Exceptions: DaisyUI's default text-button component height is 40px. This is an existing component size, is a multiple of 4, and must not be replaced by a custom height.

## Typography

Only the message and action-label roles render on this phase's surface. The supporting-text row records the narrow default if future explanatory body text is separately approved; this phase introduces none.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Action label | 14px | 600 | 1.5 |
| Supporting UI text | 16px | 500 | 1.5 |
| Loading message | 18px | 500 | 1.5 |

Use exactly these two weights (500 and 600) on the blocking surface. Do not promote the loading message into a heading or display style.

## Color

Phase 1 introduces no new page palette. Retain the app's semantic DaisyUI distribution and the current modal-specific overlay exception.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--color-base-100)` | Existing application backgrounds and surfaces; unchanged beneath the overlay |
| Secondary (30%) | `var(--color-neutral)` | Existing navigation and neutral chrome; unchanged |
| Accent (10%) | `var(--color-primary)` | Existing primary application CTAs, active selections, and primary focus treatments only; not used by Cancel |
| Destructive | `var(--color-error)` | Destructive actions only; not used by request cancellation |
| Blocking overlay exception | `rgba(0, 0, 0, 0.70)` | Existing full-viewport loading mask |
| Blocking foreground exception | `#ffffff` | Spinner, loading message, Cancel label, and Cancel focus outline |

Accent reserved for: existing primary CTAs, active selections, and primary focus treatments. The conditional Cancel action is neutral because it abandons a client wait without deleting or committing data.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | None in Phase 1; this Structure phase exposes no product action |
| Conditional blocker action | `Cancel` |
| Loading message fallback | `Processing...` (unchanged) |
| Empty state heading | Not applicable; no data surface or empty state is introduced |
| Empty state body | Not applicable; no next-step empty-state copy is introduced |
| Error state | No new copy. Ordinary API failures retain existing SDK/toast handling; accepted cancellation renders no error |
| Destructive confirmation | None. Cancel is not destructive and requires no confirmation |

Do not substitute `Stop`, `Stop generation`, or `Cancel request`: those labels could imply cooperative server-side termination that this milestone does not provide.

## UI Considerations

Applicable state considerations resolved: 7/7 explicit, 0 backstop, 0 unresolved.

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| empty | Global blocking surface | covered | When no blocking state exists, the global blocker is absent and renders no placeholder. |
| loading | Global blocking surface | covered | An active selected blocker renders the existing centered spinner and its current message, plus Cancel only when that exact state is cancelable. |
| error | Global blocking surface | covered | Ordinary request failures retain existing toast handling and exact state cleanup; accepted cancellation renders no error UI. |
| populated | Global blocking surface | covered | The normal active state contains one spinner, one current-operation message, and at most one state-bound Cancel control. |
| overflow | Blocking message | covered | The message wraps and reflows inside the viewport without clipping, horizontal overflow, or truncation. |
| long-text | Blocking message | covered | Unusually long messages wrap across lines while the spinner, message, and optional control remain centered and usable. |
| long-text | Conditional Cancel control | covered | The action has fixed visible copy and accessible name `Cancel`; callers cannot replace it with longer text. |
