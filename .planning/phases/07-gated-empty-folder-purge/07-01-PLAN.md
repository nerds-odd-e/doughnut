## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → POST /api/notebooks/{notebook}/health/fix | Authenticated session; body flag; notebook path must be owner/write authorized |
| Fix → folder hard-delete | Mutates notebook structure; must not trust client folder IDs or weaken auth vs lint |
| Lint vs Fix | Lint remains report-only; mutation only on dedicated fix path |
