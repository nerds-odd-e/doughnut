# Phase 7 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Sources locked:** `07-CONTEXT.md` (D-01–D-17), `07-RESEARCH.md` (discretion resolutions), `05-UI-SPEC.md` / `06-UI-SPEC.md` (Health chrome + action bar baseline), live `NotebookHealthPanel.vue`. Mode `--auto`: no user questions; CONTEXT discretion + RESEARCH resolutions applied.

**Relationship to Phases 5–6:** This contract **extends** Health chrome. Do **not** reinvent tabs, panel shell, findings, idle copy, Run lint primary styling, Save as defaults ghost styling, spacing/typography/color tokens, or checkbox prefill. Only the gated **Fix** control and its post-fix re-lint UX are new this phase.

## Spacing Scale

Inherited from Phase 5/6 (multiples of 4), aligned to notebook Settings / Health panels:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tight inline gaps only if needed |
| sm | 8px | Action-bar control gaps (`gap-2`) between Run lint, checkbox, Fix, and Save as defaults |
| md | 16px | Nested collapse indent / content padding inside collapse (unchanged) |
| lg | 24px | Panel section padding (`p-6`), section/stack gaps (`mb-6`) (unchanged) |
| xl | 32px | Not required for Health panel |
| 2xl | 48px | Not required for Health panel |
| 3xl | 64px | Not required for Health panel |

**Exceptions:** Collapse titles keep Phase 5 `py-3` (12px vertical) where findings render — unchanged.

**Layout shell:** Health panel remains in the same `container mx-auto … max-w-6xl` notebook page. Action bar remains `flex flex-wrap items-center gap-2` (or `gap-4`) at top of panel section — **four** controls must wrap cleanly on narrow widths without inventing a second row layout system.

**Action-bar density:** Keep `daisy-btn-sm` on all buttons (Run, Fix, Save). Do not enlarge Fix relative to siblings. Do not add icons, badges, or a count chip separate from the button label. Gaps stay `gap-2` (8px) so the bar does not feel sparse or cramped when Fix appears.

## Typography

Exactly three sizes and two weights — **identical to Phase 5/6**:

| Role | Size | Weight | Line Height | Maps to |
|------|------|--------|-------------|---------|
| Body | 14px | 400 | 1.5 | Idle prompt, empty-group message, leaf finding `label` (`text-sm`) |
| Label | 14px | 600 | 1.2 | Collapse group headers (`text-sm font-semibold`) |
| Heading | 18px | 600 | 1.2 | Optional panel section title if used (`text-lg font-semibold`); tabs use existing `daisy-tab-lg` |

Do **not** add a fourth size or display type. Fix control label uses DaisyUI `daisy-btn-sm` default label typography (same family as Run lint / Save), not a new type role.

**Severity:** Omit severity badges and severity text (inherited Phase 5).

## Color

DaisyUI semantic tokens only (no new hex palette). Ratio relative to Health panel surfaces — **identical to Phase 5/6**, with Fix explicitly **not** primary accent:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `base-100` | Health panel section background (`bg-base-100`) |
| Secondary (30%) | `base-200` / `base-200/50` | Tab strip, collapse surfaces, borders `border-base-300` |
| Accent (10%) | `primary` | **Run lint** primary button only (`daisy-btn daisy-btn-primary daisy-btn-sm`) |
| Destructive | not used on the action bar | Do **not** use `daisy-btn-error` for Fix in v1 — reserve destructive-looking treatment for true delete dialogs elsewhere in the product |

**Accent reserved for:** `Run lint` primary button only (unchanged from Phase 5).

**Not accent:** Fix (Remove N empty folders), Save as defaults, tab active state, checkbox, collapse chrome, leaf labels, borders.

**Fix control color:** Secondary action — `daisy-btn daisy-btn-secondary daisy-btn-sm`. Must **not** use `daisy-btn-primary` (primary stays on Run lint). Must **not** use `daisy-btn-ghost` (that remains Save as defaults). Must **not** use `daisy-btn-error` / heavy destructive styling that fights the action bar.

**Disabled Fix:** Native `disabled` on the button (DaisyUI disabled button styles). Visible but not clickable; no opacity-only “fake disabled” without the `disabled` attribute.

**Text:** `text-base-content` for headers; `text-base-content/70` for idle/secondary body (unchanged).

## Interaction Contract

### Inherited (Phases 5–6 — do not regress)

| Area | Contract |
|------|----------|
| Tab shell | Health notebook-only; open Health alone does **not** call lint, Fix, or mutate notebook data |
| Run lint | Bodyless `POST .../health/lint` via `apiCallWithLoading`; report-only; checkbox never changes lint request |
| Prefill / Save | Phase 6: checkbox from `currentUser.healthRemoveEmptyFoldersDefault`; Save PATCHes user preference only — never calls Fix |
| Findings | Expandable groups from wire shape; labels text-only; no client regroup |
| Action bar visibility | Present in both idle and report views so Re-Run / Fix / Save remain available |

### Fix enablement (AFIX-02, AFIX-03, D-01–D-02)

| Condition | Fix button state |
|-----------|------------------|
| **Remove empty folders** unchecked | **Disabled** (visible) |
| Checkbox checked, no successful report yet (`report === null`) | **Disabled** (visible); treat empty-folder count as `0` |
| Checkbox checked, last report has `empty_folders` group with **0** items (or group absent) | **Disabled** (visible) |
| Checkbox checked **and** last successful report has `empty_folders` items length **≥ 1** | **Enabled** |
| Idle view (no findings) | Fix remains on the action bar, **disabled** — gate stays discoverable |

**Count source:** `N = report.groups.find(g => g.ruleId === "empty_folders")?.items?.length ?? 0`. Do **not** count nested children of other rules. Do **not** include `readme_only_folders`.

**Discoverability:** Always render the Fix control on the Health action bar (idle and findings). Never hide it when disabled.

### Fix click / purge flow (D-04, D-05, D-13, D-14)

| State | Behavior |
|-------|----------|
| Click when enabled | **No confirm dialog.** Immediately call `apiCallWithLoading(() => NotebookHealthController.fix({ path: { notebook }, body: { removeEmptyFolders: true } }))`. Thin bar only (no `blockUi`). Do **not** send user defaults or folder IDs. |
| Click when disabled | No-op (native disabled) |
| Fix in flight | Global thin loading bar; do not invent a full-panel spinner |
| Fix success | Automatically re-run bodyless lint (same path as **Run lint**) and replace the panel report with the fresh result. Purged fully empty folders disappear from findings; readme-only / dead-link groups remain as applicable. |
| Fix failure | Keep prior report visible; error via existing `apiCallWithLoading` toast — no inline error panel; no optimistic clearing of findings. User can retry after addressing the gate (or re-Run lint). |
| Partial success UI | **Forbidden** — never claim purge success until fix succeeds; never replace report until re-lint succeeds after fix |

### Action bar layout (Phase 7)

| Control | Placement | Classes | Phase 7 behavior |
|---------|-----------|---------|------------------|
| Run lint | Leading (primary) | `daisy-btn daisy-btn-primary daisy-btn-sm` | Triggers lint only |
| Remove empty folders | Same bar, after Run | existing `CheckInput` wrapper | Prefill + run-scoped gate for Fix |
| Fix (Remove N empty folders) | Same bar, **after checkbox, before Save as defaults** | `daisy-btn daisy-btn-secondary daisy-btn-sm` | Gated purge; then auto re-lint |
| Save as defaults | Trailing on bar | `daisy-btn daisy-btn-ghost daisy-btn-sm` | Persists checkbox preference only |

Layout: `flex flex-wrap items-center gap-2` at top of panel — order **Run lint → Remove empty folders → Fix → Save as defaults**. Visible in both idle and report views.

**Visual hierarchy rule:** Primary accent stays exclusively on **Run lint**. Fix is secondary (purge intent, quieter than primary). Save remains ghost (quietest). Density matches Phase 5/6 `btn-sm` + `gap-2`; do not add a second action row or overflow menu.

**Self-check coverage (required):**

| Concern | Contract location |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Tab label | Health *(inherited)* |
| Primary CTA | Run lint *(inherited)* |
| Fix option checkbox | Remove empty folders *(inherited)* |
| Fix CTA (count known, N ≥ 1) | Remove N empty folders — e.g. `Remove 3 empty folders` where N is the `empty_folders` item count from the last report |
| Fix CTA (count unavailable or N = 0) | Remove empty folders — fallback when no report or zero empty-folder findings; button still disabled when N = 0 |
| Save preference CTA | Save as defaults *(inherited)* |
| Idle state body | Run lint to check this notebook for empty folders, readme-only folders, and dead wiki links. *(inherited)* |
| Empty group (expanded, count 0) | No findings *(inherited)* |
| Error state (Fix, Run, or Save) | Global toast from `apiCallWithLoading` (existing product error path). No custom inline error copy required. User can retry **Remove N empty folders**, **Run lint**, or **Save as defaults**. |
| Destructive confirmation | **none** — no confirm dialog in v1. Consent = checked **Remove empty folders** + findings preview (`empty_folders` list). |

**Do not use** as the only Fix label: “Fix”, “Apply”, “Apply fix”, or “Purge” alone. Prefer the explicit count form whenever N ≥ 1.

**Do not** invent success toast copy beyond existing `apiCallWithLoading` behavior; the refreshed findings list is the success signal.

**Group titles** come from the API — unchanged from Phase 5.

## UI Considerations

> Shape-rooted UI state coverage for planner lift (`must_haves`). Empty/error **copy** is in Copywriting Contract; this section references those rows.

Applicable state considerations resolved: **9 covered, 1 backstop, 0 unresolved** (Phase 7 delta elements; Phase 5/6 considerations remain in force via inheritance)

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| empty | Fix control when no empty_folders | ✅ covered | Button visible with fallback label **Remove empty folders**, native `disabled`. No separate empty-state message for Fix. |
| empty | Health findings before first Run | ✅ covered | Inherited idle prompt; Fix stays disabled on the action bar. |
| loading | Fix interactive-control | ✅ covered | In-flight uses existing thin-bar `apiCallWithLoading` only — no panel skeleton, no `blockUi` modal. Post-success re-lint uses the same thin-bar path as Run lint. |
| error | Fix / post-fix re-lint | ✅ covered | Failures surface via global toast; prior report remains; no optimistic clear. User retries Fix or Run lint. |
| populated | Fix enabled happy path | ✅ covered | Checkbox on + N ≥ 1 → enabled label **Remove N empty folders**; click purges then refreshes findings so empty folders are gone. |
| partial | Report with mixed groups after Fix | ✅ covered | Re-lint may still show readme-only and/or dead links while empty_folders is empty — normal; Fix returns to disabled when empty_folders count is 0. |
| overflow | Action bar with four controls | 🧪 backstop | `flex-wrap` + `gap-2` + `btn-sm` keep Run, checkbox, Fix, and Save reachable on narrow widths. Verify visually that Fix remains tappable without horizontal clip. |
| zero-one-many | Fix label N | ✅ covered | N = 0 → fallback label + disabled; N = 1 → `Remove 1 empty folders`; N = many → `Remove {N} empty folders`. Singular wording not required (locked explicit pattern uses “folders”). |
| long-text | Fix button label | ✅ covered | Label length scales with digit count only (`Remove N empty folders`); wraps with action bar `flex-wrap` if needed — no ellipsis truncation required. |

## Out of Scope (UI)

Do not design or implement in Phase 7:

- Per-folder multi-select delete UI
- Confirm / undo / recycle-bin dialogs for purge
- Auto-fix controls for dead wiki links or readme-only folders
- Primary or error styling on Fix that steals accent from Run lint
- Hiding Fix when disabled (must stay visible for discoverability)
- Sending user defaults or folder IDs on the fix request
- Severity badges, click-through labels, `/health` route, findings dialog
- Health on folder settings
- New design tokens, themes, or icon sets
