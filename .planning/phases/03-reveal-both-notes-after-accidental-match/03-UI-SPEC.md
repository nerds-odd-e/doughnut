# Phase 3 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Sources locked:** `03-CONTEXT.md` (D-01–D-06), `03-RESEARCH.md`, live `AnsweredSpellingQuestion.vue` / `NoteShow.vue` / DaisyUI recall alerts. Mode `--auto`: no interactive prompts; discretionary choices logged below.

**Product locks (do not contradict):**
- **D-03:** Full `NoteShow` per matched note (`:expand-children="false"`)
- **D-04:** Vertical stack — reviewed first, then labeled **Matched note(s)** section
- **D-05:** Distinct `ACCIDENTAL_MATCH` alert; non-success styling; plain wrong unchanged
- **D-06:** No add-link / `LinkInsertionChoice` / preselection UI this phase

## Spacing Scale

Declared values (multiples of 4), aligned to recall result + Tailwind spacing used nearby:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline tight gaps only if needed |
| sm | 8px | Compact gaps inside existing note chrome |
| md | 16px | Gap between successive matched `NoteShow`s (`gap-4` / `space-y-4`); padding under section heading |
| lg | 24px | Top margin before **Matched note(s)** section (`mt-6`) — separates reviewed block from matches |
| xl | 32px | Not required for this surface |
| 2xl | 48px | Not required for this surface |
| 3xl | 64px | Not required for this surface |

**Exceptions:** Existing `NoteUnderQuestion` chrome keeps its live padding/`margin-top: 15px` — do not restyle that component in this phase. New Phase 3 spacing (matched section) must stay on the 4/8/16/24 scale above.

**Layout shell:** Single-column vertical stack inside the existing recall result mount (same width as today’s spelling answer result). No max-width change, no grid/columns.

## Typography

Exactly three sizes and two weights, taken from recall / AssimilationSettings section patterns:

| Role | Size | Weight | Line Height | Maps to |
|------|------|--------|-------------|---------|
| Body | 16px | 400 | 1.5 | Alert message text (`strong` inside `daisy-alert`); default reading text |
| Label | 14px | 600 | 1.2 | Defensive empty one-liner under alert only if shown (`text-sm font-semibold`) |
| Heading | 18px | 600 | 1.2 | **Matched note(s)** section title (`text-lg font-semibold`) — same emphasis as `AssimilationSettings` section headers |

Do **not** add a fourth size. Do **not** use the global bare `h3 { font-size: 1.5rem }` (24px) from `daisyui.css` for the matched section — that breaks the three-size contract. Implement the section title with explicit `text-lg font-semibold` (18px/600), not an unstyled `h3`.

**NoteShow interiors** keep their existing typography (out of scope to restyle).

**Weights used:** 400 (regular) and 600 (semibold) only.

## Color

DaisyUI semantic tokens only (no new hex palette). Ratio relative to the spelling answer-result surface:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `base-100` | Result surface / page background behind the stack |
| Secondary (30%) | `base-200` / `border-base-300` | Existing `NoteUnderQuestion` chrome (`bg-base-200 border-base-300`); borders only — not a new card shell around matches |
| Accent (10%) | `primary` | Reserved exclusively for the future Phase 4 add-link primary control — **zero Phase 3 usages** |
| Destructive / miss | `error` (`daisy-alert-error`) | Both plain incorrect and `ACCIDENTAL_MATCH` alerts (non-success; D-05) |
| Success | `success` (`daisy-alert-success`) | Correct answers only (unchanged) |

**Accent reserved for:** Phase 4 add-link primary CTA only (deferred). In Phase 3 the executor must **not** apply `daisy-btn-primary`, accent badges, or accent-tinted chrome on the reveal stack.

**Not accent:** Accidental-match alert, plain-incorrect alert, **Matched note(s)** heading, `NoteShow`, `NoteUnderQuestion`, memory-tracker link.

**Alert styling (D-05):**
- Correct → `daisy-alert-success` (unchanged)
- Plain wrong (`correct === false`, outcome ≠ `ACCIDENTAL_MATCH`) → `daisy-alert-error` (unchanged)
- Accidental match (`outcome === 'ACCIDENTAL_MATCH'`) → `daisy-alert-error` (non-success; same miss family as plain wrong — distinction is **copy**, not a success tint)

**Text:** `text-base-content` for section heading; alert text inherits DaisyUI alert content color.

## Interaction Contract

### Alert branch (D-05)

| Condition | Alert class | Copy source |
|-----------|-------------|-------------|
| `answer.correct === true` | `daisy-alert-success` | `Correct!` (unchanged) |
| `correct === false` and `outcome !== 'ACCIDENTAL_MATCH'` | `daisy-alert-error` | `Your answer \`${spellingAnswer}\` is incorrect.` (exact; unchanged) |
| `outcome === 'ACCIDENTAL_MATCH'` | `daisy-alert-error` | Accidental-match string below (distinct) |

### Vertical stack (D-03, D-04)

Order (top → bottom), single column:

1. Result alert  
2. `NoteUnderQuestion` (reviewed)  
3. `ViewMemoryTrackerLink`  
4. Reviewed `NoteShow` (`expand-children=false`)  
5. **If** accidental match with ≥1 `matchedNotes`: section **Matched note(s)** then one `NoteShow` per match (`expand-children=false`), id-ascending order from props  

### Matched list states

| State | Behavior |
|-------|----------|
| Plain wrong / correct | Matched section **absent** (no empty heading) |
| `ACCIDENTAL_MATCH` + non-empty `matchedNotes` | Section visible; render all entries |
| `ACCIDENTAL_MATCH` + empty/missing `matchedNotes` (defensive; should not happen per D-02) | Omit section; keep accidental-match alert (see Empty copy) |
| Multiple matches | Stacked `NoteShow`s with `md` (16px) vertical gap; no pagination/collapse |
| `NoteShow` loading | Existing `NoteRealmLoader` / `ContentLoader` per instance — no new skeleton system |
| `NoteShow` load failure | Keep alert + sibling notes visible; inherit existing NoteShow/error handling — no new retry CTA |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | **none** — Phase 3 is reveal-only. Do not invent Continue / OK / Next / Submit buttons. (Phase 4 owns add-link CTA.) |
| Accidental-match alert | `Your answer \`${spellingAnswer}\` names another note — not correct for this review.` |
| Plain incorrect alert | `Your answer \`${spellingAnswer}\` is incorrect.` (unchanged) |
| Correct alert | `Correct!` (unchanged) |
| Matched section heading | `Matched note(s)` |
| Empty state heading | (none — no empty panel chrome) |
| Empty state body | When `ACCIDENTAL_MATCH` but `matchedNotes` missing/empty: keep the accidental-match alert only; do **not** show a “No data found” / “No results” / “Nothing here” panel. Optional muted one-liner under the alert if a test needs an observable: `Matched note details are unavailable.` |
| Error state | Outcome miss messaging = alert rows above (problem + meaning). Note load failures: inherit existing NoteShow path; user can continue the recall session. No new “Something went wrong” string. |
| Destructive confirmation | none — no destructive actions in Phase 3 |

**Auto-logged copy decision (discretion):** Accidental-match alert uses “names another note — not correct for this review” so it is visibly different from the plain “is incorrect” string while remaining a miss for the reviewed note.

| overflow | NoteShow (E3) | 🧪 backstop | NoteShow content scrolls within existing note layout; verify long note bodies do not break the single-column result stack. |
