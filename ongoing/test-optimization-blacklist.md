`e2e_test/features/circles/creating_circles.feature` — New user via circle invitation — ~3468ms — genuine new-user invitation journey: logout → full-load invitation/login page → identify as new user (`invoke('val')`+`trigger('input')`) → create profile (`assignValue` already uses `invoke('val')`) → API `POST /api/circles/join` + full-load circle page → assert 2 members; the two full page loads are necessary (logged-out invitation link load, and post-join circle page render for the UI "two members" assertion); the post-join circle visit can't be cleanly SPA-nav'd because `I visit the invitation link` uses a full origin URL (not a `routerPush` path), so `@firstVisited` never becomes `'yes'` and the first `routerPush` still full-loads — proposed 2026-07-23
